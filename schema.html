JSON Schema Reference

This page documents every field in the Pillar program JSON format. Programs authored to this schema can be imported into the Pillar iOS app via Programs → Import, or hosted at a URL for one-tap updates. Use the Program Builder to create and export programs without writing JSON by hand.

Keeping this in sync: whenever a field is added or changed in ProgramPayload.swift in the Pillar iOS app, update this page and the validate_programs.py script's allowed-value constants in the same commit.
Root object

Wrapper

Every file starts with this envelope. A single file may contain multiple programs, though one per file is the convention.

{
  "schemaVersion": "1.0",
  "programs": [ /* ...Program */ ]
}
FieldTypeRequiredDescription
schemaVersionstring✅Always "1.0".
programsProgram[]✅One or more Program objects.
Object

Program

Top-level program metadata and all its content.

{
  "id": "four_pillar",
  "name": "Four Pillar Training Program",
  "version": "1.15.0",
  "baseId": null,
  "upgradeFromVersion": null,
  "description": "...",
  "tags": ["home", "dumbbell", "longevity"],
  "targetProfile": "intermediate",
  "createdAt": "2024-01-01",
  "sourceUrl": "https://jalsemgeest.github.io/pillar-programs/four_pillar_v1.json",
  "releaseNotes": "v1.15.0: added goal text to all plans.",
  "plans": [ /* ...Plan */ ],
  "exercises": [ /* ...Exercise */ ],
  "mobilityRoutines": [ /* ...MobilityRoutine */ ],
  "benchmarks": [ /* ...Benchmark */ ],
  "citations": [ /* ...Citation */ ]
}
FieldTypeRequiredDescription
idstring✅Stable identifier. Never change after publishing — the app uses it to match bundled upgrades to stored copies. Use snake_case.
namestring✅Display name shown in the Programs list.
versionstring✅Semantic version "MAJOR.MINOR.PATCH". The app auto-upgrades a stored program when the bundled version is newer. Increment MINOR for content changes, PATCH for corrections.
baseIdstring—If this program is a fork of another, the original program's id. Informational only.
upgradeFromVersionstring—Minimum version of the base program this upgrade applies to. Informational only.
descriptionstring✅Full description shown on the program detail screen. Plain text — markdown is not rendered.
tagsstring[]✅Free-form tags for filtering (e.g. "home", "dumbbell", "time-efficient"). May be [].
targetProfilestring✅Intended audience (e.g. "beginner", "intermediate", "advanced"). Informational only.
createdAtstring✅ISO 8601 date the program was first authored ("YYYY-MM-DD").
sourceUrlstring—Canonical URL where the app checks for updates. When present, the "Check for update" flow uses this URL instead of a hardcoded fallback.
releaseNotesstring—Human-readable change summary for the current version.
plansPlan[]✅One or more Plan objects.
exercisesExercise[]✅Program-specific exercise definitions. May be []. See Exercise.
mobilityRoutinesMobilityRoutine[]✅Daily habits / warm-up / cool-down routines. May be []. See MobilityRoutine.
benchmarksBenchmark[]✅Performance benchmarks tracked on the Home screen. May be []. See Benchmark.
citationsCitation[]—Research citations referenced by exercises or the program description. See Citation.
Object

Plan

A plan is one variant of the program (e.g. "4-Day", "3-Day", "Home"). A user selects one active plan at a time. Plans support optional periodization phase metadata for block-based programs.

{
  "id": "four_day",
  "name": "4-Day Plan",
  "scheduleDescription": "Mon · Wed · Thu · Sat",
  "daysPerWeek": 4,
  "orderIndex": 0,
  "phaseName": "Hypertrophy",
  "phaseOrder": 2,
  "phaseDurationWeeks": 10,
  "deloadWeekNumber": 10,
  "dynamicEquipment": false,
  "goal": "Build muscle mass with progressive overload...",
  "workouts": [ /* ...Workout */ ]
}
FieldTypeRequiredDescription
idstring✅Stable identifier within this program.
namestring✅Display name (e.g. "4-Day Plan", "Foundation").
scheduleDescriptionstring✅Human-readable schedule shown on the plan card (e.g. "Mon · Wed · Thu · Sat").
daysPerWeekint✅Number of training days per week.
orderIndexint✅0-based sort position within the program's plan list.
workoutsWorkout[]✅Ordered list of Workout sessions.
phaseNamestring—Periodization phase name (e.g. "Foundation", "Hypertrophy", "Strength", "Conditioning"). Omit for non-phased programs.
phaseOrderint—1-based position of this phase in the program sequence.
phaseDurationWeeksint—Intended phase length in weeks. Drives the phase progress bar on the Home screen.
deloadWeekNumberint—Which week (1-based) is the deload week. The app shows an amber "DELOAD" badge and suppresses phase-themed mobility routines that week.
dynamicEquipmentbool—When true, the user picks their equipment at workout start and exercises auto-swap to compatible alternatives. Default: false.
goalstring—Short description of the plan's training intent. Shown in the "Program Info" sheet accessible from the three-dot menu during an active workout.
Object

Workout

A single training session within a plan.

{
  "id": "four_day_mon",
  "dayLabel": "Monday",
  "sessionType": "Lower Body",
  "durationRange": "50–65 min",
  "location": "Home",
  "keyMetrics": ["Trap bar deadlift weight", "Hip thrust weight"],
  "orderIndex": 0,
  "activityType": "strength",
  "preMobilityIds": ["morning_cars"],
  "postMobilityIds": [],
  "groups": [ /* ...WorkoutGroup */ ]
}
FieldTypeRequiredDescription
idstring✅Stable identifier within this plan.
dayLabelstring✅Suggested training day (e.g. "Monday", "Day A"). The user can override this in schedule settings.
sessionTypestring✅Session name displayed during the workout (e.g. "Lower Body", "Hinge + Horizontal").
durationRangestring✅Estimated duration shown on the workout card (e.g. "50–65 min").
locationstring✅Where the workout takes place (e.g. "Home", "Gym"). Informational.
keyMetricsstring[]✅Exercises to track as primary progress indicators. Shown as a subtitle on the workout card. May be [].
orderIndexint✅0-based sort position within the plan.
groupsWorkoutGroup[]✅Ordered list of WorkoutGroup blocks. Also accepted as "blocks" for legacy compatibility.
activityTypestring—Maps to an Apple Health workout type. See enum values. Omit to let the app infer from exercise categories.
preMobilityIdsstring[]—IDs of MobilityRoutine objects shown as a checklist before the first exercise.
postMobilityIdsstring[]—IDs of MobilityRoutine objects shown as a checklist after the last exercise.
Object

WorkoutGroup

A named block of exercises within a workout. Groups appear as section headers in the workout flow. Use group names to communicate intent (e.g. "SUPERSET A", "CORE FINISHER", "HINGE — PRIMARY").

{
  "name": "HINGE — PRIMARY",
  "orderIndex": 0,
  "exercises": [ /* ...PrescribedExercise */ ]
}
FieldTypeRequiredDescription
namestring✅Display name for this block.
orderIndexint✅0-based sort position within the workout.
exercisesPrescribedExercise[]✅Ordered list of PrescribedExercise entries.
Object

PrescribedExercise

A single exercise prescription within a workout group.

{
  "exerciseId": "db_rdl",
  "sets": 3,
  "repsMin": 5,
  "repsMax": 6,
  "repsText": null,
  "load": "Heavy — RPE 8 by set 3",
  "tempo": "30X0",
  "notes": "Push hips back, feel hamstring stretch.",
  "priority": "core",
  "orderIndex": 0,
  "logMode": "reps",
  "durationSeconds": null,
  "perSide": false,
  "restSeconds": 90,
  "showHeartRateZones": false,
  "targetZoneLow": null,
  "targetZoneHigh": null,
  "timerStartMode": "immediate",
  "alternatives": [ /* ...ExerciseAlternative */ ],
  "citationIds": ["schoenfeld_2019"]
}
FieldTypeRequiredDescription
exerciseIdstring✅Must match an id in the app's ExerciseDatabase.json or this program's exercises array. Never change after publishing — the app links logged history to prescriptions by this ID.
setsint✅Number of working sets.
repsMinint—Minimum of the rep range (e.g. 8 in "8–10 reps"). Omit for time-based or distance-based exercises.
repsMaxint—Maximum of the rep range. When equal to repsMin, the app displays a single number ("8 reps").
repsTextstring—Free-form prescription displayed verbatim (e.g. "40m", "AMRAP", "Max hold"). Do not include the set count — the app prepends "N sets × " automatically.
loadstring✅Weight/load guidance (e.g. "70–80% 1RM", "Bodyweight", "Heavy — RPE 8"). Use "" if no guidance applies.
tempostring—Tempo in "ECCE" notation (eccentric · pause-bottom · concentric · pause-top, e.g. "30X0"). "X" = explosive. Shown as a badge on the exercise card.
notesstring—Coaching cue shown below the exercise name during the workout.
prioritystring✅"core" = always performed; "optional" = shown with a badge and skippable. Any unrecognised value is treated as "core".
orderIndexint✅0-based sort position within the group.
logModestring—How sets are logged. "reps" (default), "timer" (countdown from durationSeconds), or "duration" (count-up stopwatch). See enum values.
durationSecondsint—Target duration in seconds. Required when logMode is "timer". Optional for "duration" (sets a suggested target).
perSidebool—When true, the set is performed on each side separately. The rep count or timer applies per side, with a 5-second switch break between sides. Default: false.
restSecondsint—Rest period in seconds after each set. Overrides the app default of 90 s.
showHeartRateZonesbool—When true, a live HR zone bar is displayed below the rest timer. Requires Apple Watch streaming HR. Default: false.
targetZoneLowint—Lower bound of the target HR zone band (1–5). Used with showHeartRateZones and timerStartMode: "waitForZone".
targetZoneHighint—Upper bound of the target HR zone band. Omit to target a single zone (targetZoneLow only).
timerStartModestring—"immediate" (default): countdown starts when the user taps Go. "waitForZone": waits until live HR enters the target zone, then auto-starts. Only meaningful with logMode: "timer" and targetZoneLow set.
alternativesExerciseAlternative[]—Curated substitutions shown in the exercise swap sheet. Must be objects — not bare strings. See ExerciseAlternative.
citationIdsstring[]—IDs of Citation objects supporting this prescription.
Object

ExerciseAlternative

A curated substitution for a prescribed exercise, shown in the exercise swap sheet.

Alternatives must be objects, not bare strings. "alternatives": ["goblet_squat"] will fail to decode — use "alternatives": [{"exerciseId": "goblet_squat"}] instead.
{
  "exerciseId": "dumbbell_floor_press",
  "context": "No bench available",
  "equipmentTags": ["dumbbell"],
  "isIdeal": true
}
FieldTypeRequiredDescription
exerciseIdstring✅ID of the alternative exercise. Must exist in ExerciseDatabase.json or this program's exercises array.
contextstring—Human-readable note explaining when to use this alternative (e.g. "No bench available"). Shown in the substitution sheet.
equipmentTagsstring[]—Equipment required for this alternative (e.g. ["dumbbell", "bench"]). Used to filter by the user's equipment profile. Omit or use [] for bodyweight.
isIdealbool—false = lower-stimulus fallback; the app shows a "not ideal" badge. Default (omit): true.
Object

Exercise (program-level definition)

Program-specific exercise definitions providing coaching notes and citation links. The exerciseId on a PrescribedExercise may reference either this array or the global exercise database — this array is checked first.

{
  "id": "trap_bar_deadlift",
  "name": "Trap Bar Deadlift",
  "category": "hinge",
  "equipment": ["trap_bar"],
  "notes": "Stand in the centre of the trap bar. Push the floor away...",
  "citationIds": ["schoenfeld_2017"]
}
FieldTypeRequiredDescription
idstring✅Unique identifier. Must match the exerciseId used in prescriptions.
namestring✅Display name.
categorystring✅Movement pattern. See enum values.
equipmentstring[]✅Equipment required (e.g. ["dumbbell"], ["barbell", "rack"], [] for bodyweight).
notesstring—Full coaching notes / technique description.
citationIdsstring[]—IDs of Citation objects supporting this exercise's selection.
Object

MobilityRoutine

A mobility or habit routine shown on the Home screen Daily Habits card, or as a pre/post-workout checklist within a session. Routines support a phase-aware rotation system via themeTag and phaseNames.

{
  "id": "morning_cars",
  "name": "Morning CARs Routine",
  "timing": "general",
  "durationDescription": "5–7 min",
  "notes": "Do before coffee. Daily consistency beats occasional long sessions.",
  "themeTag": null,
  "phaseNames": null,
  "exercises": [ /* ...MobilityExercise */ ]
}
FieldTypeRequiredDescription
idstring✅Stable identifier. Referenced by preMobilityIds / postMobilityIds on workouts.
namestring✅Display name.
timingstring✅When this routine is shown. See enum values.
durationDescriptionstring✅Estimated time shown on the routine card (e.g. "5–7 min").
notesstring—Coaching note or intent description shown on the routine card.
themeTagstring—Mobility rotation tag. null = anchor routine (always shown). "A"–"F" = phase-themed, shown only when the user's active phase matches phaseNames.
phaseNamesstring[]—Phase names this themed routine applies to (must match phaseName values on plans). null = applies to all phases. Only meaningful when themeTag is set.
exercisesMobilityExercise[]✅Ordered list of MobilityExercise items.
Object

MobilityExercise

A single item within a mobility routine checklist.

{
  "name": "Hip CARs",
  "duration": "5 circles each direction / side",
  "frequency": "Daily",
  "focusArea": "Hip joint health",
  "notes": "Keep the pelvis neutral. Max range, min compensation.",
  "priority": "core",
  "orderIndex": 0
}
FieldTypeRequiredDescription
namestring✅Exercise name displayed on the checklist row.
durationstring✅Prescription text (e.g. "30 seconds", "3 reps / side").
orderIndexint✅0-based sort position within the routine.
frequencystring—How often to do this exercise (e.g. "Daily", "3×/week"). Informational.
focusAreastring—Body area or adaptation target (e.g. "Hip flexor length"). Informational.
notesstring—Technique or intent note.
prioritystring—"core" (always shown) or "optional" (shown with an optional badge). Default: "core".
Object

Benchmark

A performance test tracked on the Home screen benchmarks grid. Two types: exercise PRs (tracked automatically from logged sets) and standalone tests (manually logged).

{
  "id": "deadlift_pr",
  "name": "Trap Bar Deadlift 3RM",
  "description": "Best triple at competition-legal depth.",
  "type": "exercise_pr",
  "exerciseId": "trap_bar_deadlift",
  "metric": "weight",
  "unit": "lbs",
  "orderIndex": 0,
  "citationIds": [],
  "targets": [ /* ...BenchmarkTarget */ ]
}
FieldTypeRequiredDescription
idstring✅Stable identifier.
namestring✅Display name.
descriptionstring—Additional context shown in the benchmark detail view.
typestring✅"exercise_pr" — tracks the best logged set for a specific exercise. "standalone_test" — manually logged results (e.g. VO₂ max, resting HR).
exerciseIdstring—Required when type is "exercise_pr". Must match an exercise ID.
metricstring✅What is measured. See enum values.
unitstring✅Display unit (e.g. "lbs", "kg", "seconds", "ml/kg/min").
orderIndexint✅0-based sort position on the Home benchmarks grid.
citationIdsstring[]—IDs of supporting Citation objects.
targetsBenchmarkTarget[]✅Reference points shown as markers on the progress chart. May be []. See BenchmarkTarget.
Object

BenchmarkTarget

A labelled reference value on a benchmark (e.g. population norms, personal goals).

{ "label": "Novice", "value": 135.0, "orderIndex": 0 }
FieldTypeRequiredDescription
labelstring✅Target label (e.g. "Baseline", "Goal", "Elite").
valuenumber✅Numeric target in the benchmark's unit.
orderIndexint✅0-based sort position within the target list.
Object

Citation

A research reference linked to exercises or benchmarks via citationIds.

{
  "id": "schoenfeld_2019",
  "principle": "Minimum effective volume for hypertrophy",
  "authors": "Schoenfeld BJ, Ogborn D, Krieger JW",
  "journal": "Journal of Strength and Conditioning Research",
  "year": 2019,
  "keyFinding": "Even a single set per muscle group per session produces significant hypertrophy.",
  "url": "https://doi.org/10.1519/JSC.0000000000002585"
}
FieldTypeRequiredDescription
idstring✅Stable identifier referenced by citationIds on exercises and benchmarks.
principlestring✅Short title of the training principle this paper supports.
authorsstring✅Author(s) in short form (e.g. "Schoenfeld BJ et al.").
journalstring✅Publication venue.
yearint✅Publication year.
keyFindingstring✅One-sentence summary of the finding as it applies to this program.
urlstring—DOI or direct link to the paper.

Enum values

All fields with a fixed set of allowed values.

logMode

ValueDescription
"reps"Weight + rep count (the default when omitted).
"timer"Countdown from durationSeconds. Auto-completes at zero. durationSeconds is required.
"duration"Count-up stopwatch — the user stops it manually (e.g. dead-hang max hold). durationSeconds is optional.

activityType

ValueApple Health type
"strength"Traditional Strength Training
"functionalStrength"Functional Strength Training
"cardio"Other Cardio
"hiit"High Intensity Interval Training
"climbing"Climbing
"core"Core Training
"mixed"Mixed Cardio

timerStartMode

ValueDescription
"immediate"Countdown starts when the user taps Go (default).
"waitForZone"Waits until live HR from Apple Watch enters the target zone band, then auto-starts.

priority (PrescribedExercise)

ValueDescription
"core"Always performed (default).
"optional"Shown with an optional badge and skippable.

timing (MobilityRoutine)

ValueDescription
"general"Shown on the Home screen Daily Habits card (always visible when the routine is active).
"pre_workout"Shown as a checklist before the first exercise in any workout that references this routine's ID.
"post_workout"Shown as a checklist after the last exercise.

category (Exercise)

ValueMovement pattern
"hinge"Hip hinge (deadlifts, RDLs, good mornings)
"squat"Knee-dominant (squats, lunges, split squats)
"push"Horizontal or vertical press
"pull"Horizontal or vertical pull
"carry"Loaded carries
"core"Trunk stability and anti-rotation
"cardio"Cardiovascular / conditioning
"mobility"Stretching, joint prep, CARs
"power"Explosive / ballistic
"other"Does not fit the above

type (Benchmark)

ValueDescription
"exercise_pr"Automatically tracks the best logged set for the linked exerciseId.
"standalone_test"Manually logged results entered by the user (e.g. VO₂ max, resting HR).

metric (Benchmark)

ValueDescription
"weight"Load in lbs or kg
"reps"Repetition count
"time"Duration in seconds
"distance"Distance in metres or miles
"score"Generic numeric score

Validation

The validate_programs.py script in the Pillar iOS repo cross-references every program JSON against the app's ExerciseDatabase.json and checks:

  • exerciseId values resolve to known exercises
  • logMode matches the exercise database's suggestedLogMode where one is set
  • logMode: "timer" exercises have durationSeconds set
  • Required fields are present on every object
  • alternatives entries are objects, not bare strings
python3 Pillar/Scripts/validate_programs.py

Minimal example

A valid program with one plan, one workout, one group, and one exercise — the smallest file the app will accept.

{
  "schemaVersion": "1.0",
  "programs": [
    {
      "id": "my_program",
      "name": "My Program",
      "version": "1.0.0",
      "description": "A simple example program.",
      "tags": ["home", "dumbbell"],
      "targetProfile": "intermediate",
      "createdAt": "2026-01-01",
      "plans": [
        {
          "id": "my_plan",
          "name": "3-Day",
          "scheduleDescription": "Mon · Wed · Fri",
          "daysPerWeek": 3,
          "orderIndex": 0,
          "workouts": [
            {
              "id": "day_a",
              "dayLabel": "Day A",
              "sessionType": "Full Body",
              "durationRange": "45–55 min",
              "location": "Home",
              "keyMetrics": ["Goblet squat weight"],
              "orderIndex": 0,
              "activityType": "strength",
              "groups": [
                {
                  "name": "SQUAT",
                  "orderIndex": 0,
                  "exercises": [
                    {
                      "exerciseId": "goblet_squat",
                      "sets": 3,
                      "repsMin": 8,
                      "repsMax": 10,
                      "load": "Moderate-heavy",
                      "priority": "core",
                      "orderIndex": 0
                    }
                  ]
                }
              ]
            }
          ]
        }
      ],
      "exercises": [],
      "mobilityRoutines": [],
      "benchmarks": [],
      "citations": []
    }
  ]
}
